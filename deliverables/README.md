# Grable Services Website Revamp Deliverables

This folder is the working control center for the revamp. Research stays in `../research/`; build-ready plans, copy skeletons, page models, and execution backlog live here.

## Current Direction

**Positioning:** Grable Services helps federal agencies and enterprise operators host, upgrade, and operate IBM Maximo, IBM Storage, and AI workloads, with SDVOSB accountability, verified IBM reselling capability, and production-grade infrastructure services.

**Visual system:** Mission-Grade Engineering: Hangar Navy, Signal Cyan, Mission Gold, Space Grotesk, Inter, JetBrains Mono.

**Build target:** Next.js App Router, Vercel, HubSpot forms, GA4, PostHog. Sanity CMS and Pagefind move to Phase 2 unless launch publishing volume requires them sooner.

## Files

- `00-launch-roadmap.md` - phased launch plan and what ships first.
- `01-site-map-and-navigation.md` - approved IA, redirects, navigation rules.
- `02-page-template-system.md` - reusable page templates and required sections.
- `03-core-page-copy-skeletons.md` - first-pass copy scaffolds for priority pages.
- `04-content-and-seo-backlog.md` - launch content queue, schema, and SEO priorities.
- `05-proof-data-request.md` - exact data/assets needed from the business.
- `06-engineering-backlog.md` - implementation backlog for the Next.js build.
- `10-implementation-task-tracker.md` - current implementation status, verification results, and launch blockers.
- `11-menu-item-research-tracker.md` - every current menu item with placeholder route, status, and research notes.

## Operating Rule

Claims are allowed into the structure now, but public output must stay neutral unless the source data is approved. Use `05-proof-data-request.md` for required proof and `10-implementation-task-tracker.md` to track what remains blocked.
